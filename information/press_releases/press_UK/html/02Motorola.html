Motorola Collaborates With In-Fusio To Jointly Develop A Standardized Java™ API For Mobile Games

BORDEAUX, France - May 29, 2001
Motorola and In-Fusio have signed an agreement to jointly develop a new games application programming interface (API) for mobile phones. This new client-side interface represents a milestone in the mobile games industry. The advanced API is being designed to help game developers create more compelling games for JavaTM technology-enabled mobile communication products and other wireless devices that deliver fast action and highly attractive graphics..

To compete with popular video games, mobile devices need to offer games with appealing features and an eye-catching appearance. "The market opportunity for mobile games is tremendous. Games provide a rigorous environment for testing software architectures, and they quickly raise market awareness for the entire value chain of mobile communication products," says Anne-Marie Larkin, vice president and director of Motorola's Wireless Software Applications and Services. "A unique standardized games API will give developers a consistent framework from which to write compelling, fun programs that consumers can download to Java technology-enabled handsets." .

The games API is designed to enhance the capabilities of Java technology-enabled handsets and is expected to be fully compliant with the versions of Java technology dedicated to mobile phones (J2METM, CLDC, MIDP).

Alliance of Experts

Strongly positioned on mobile entertainment, In-Fusio will contribute its knowledge of game API requirements acquired by development of ExEnR and its operational experience in game development. Motorola intends to bring its strong experience in the wireless market and in the Java programming language adapted for the wireless market, particularly in the MIDPGilles Raymond, chief executive officer at In-Fusio says, "Combining both Motorola and In-Fusio's knowledge and experience in ExEn and Java technologies will simplify the development of mobile games and accelerate the reality of this new opportunity for application developers."
